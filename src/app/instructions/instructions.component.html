How to Create a Schedule

Use the Schedule Builder to create recurring schedules with the following fields:

Minute (0-59)

Specify which minutes the schedule should run. Use comma-separated values or * for every minute.

Examples: 0 (at minute 0), 0,15,30,45 (every 15 minutes), * (every minute)

Hour (0-23)

Specify which hours the schedule should run. Use comma-separated values or * for every hour.

Examples: 9 (at 9 AM), 9,17 (at 9 AM and 5 PM), * (every hour)

Day of Month (1-31)

Specify which days of the month the schedule should run. Use comma-separated values or * for every day.

Examples: 1 (1st of month), 1,15,31 (1st, 15th, and 31st), * (every day)

Month (1-12)

Specify which months the schedule should run. Use comma-separated values or * for every month.

Examples: 1 (January), 1,6,12 (January, June, December), * (every month)

Day of Week (0-6)

Specify which days of the week the schedule should run. Sunday = 0, Monday = 1, etc. Use comma-separated values or * for every day.

Examples: 1 (Mondays), 1,5 (Mondays and Fridays), * (every day)

How the Validator Works

The Schedule Validator allows you to test cron expressions and get instant feedback:

  • Input: Enter a cron expression in the format: minute hour day month weekday
  • Validation: The system checks if the expression is properly formatted and within valid ranges
  • Results: Get immediate feedback with ✅ for valid expressions or ❌ for invalid ones
  • Description: Valid expressions show a human-readable description of when the schedule will run
  • Errors: Invalid expressions show specific error messages to help you fix issues

Accepted Input Formats

The application supports the following input formats:

Basic Examples:

  • 0,15,30,45 * * * * - Every 15 minutes
  • 0 9,17 * * 1,5 - At 9 AM and 5 PM on Mondays and Fridays
  • 30 12 1,15 * * - At 12:30 PM on the 1st and 15th of each month
  • 0 0 1 1,7 * - At midnight on January 1st and July 1st
  • * * * * * - Every minute of every hour, every day

Rules:

  • Use comma (,) to separate multiple values
  • Use * for "every" value
  • Each field must be within its valid range
  • All 5 fields are required
  • Fields must be separated by spaces

Cron Expression Task (Primary)

Make the cron expression flow work end-to-end in the Validator:

  • Form: Create a Reactive Form with a single control cronExpression (required + pattern: digits, spaces, commas, *).
  • Parse: Use ScheduleService.parseCronExpression to convert the string into a Schedule or null.
  • Validate: Use ScheduleService.validateSchedule to determine validity and collect errors/warnings.
  • Describe: If valid, show description via getHumanReadableDescription. Otherwise, show errors/warnings/format error.
  • Visibility: Only show description when valid and non-empty; only show errors/warnings when present.

Run npm test and check the Validator specs: they will stub service methods and assert your component logic is wired correctly.
